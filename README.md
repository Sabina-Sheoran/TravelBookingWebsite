## 📝 Abstract

This project focuses on designing a high-fidelity prototype of a **hotel booking web application** using **Figma**, applying core principles of **Human-Computer Interaction (HCI)**. The design emphasizes usability, visual clarity, and a smooth end-to-end user journey — from browsing hotels to completing a booking and payment.

The project also emphasizes the importance of **prototyping**, which helps designers visualize the final product before development, reducing errors and improving overall quality.

---

## 🌐 Introduction

With the rapid growth of online services, hotel booking systems have become essential in the travel and tourism industry. Users rely on these platforms to explore options, compare prices, and make reservations effortlessly. However, many existing platforms suffer from poor UI/UX, making them difficult to use.

This project uses **Figma** to create a complete, high-fidelity prototype of a hotel booking application — designed with a focus on clarity, accessibility, and smooth navigation.

## 🔍 Problem Statement

Many existing hotel booking platforms suffer from:
- Complicated navigation and cluttered layouts
- Lack of clear hotel information
- Inefficient booking processes

**This project addresses these challenges** by creating a user-friendly interface that simplifies the entire booking process while maintaining a professional and attractive design.

The system is designed to be:
- 🧭 **Easy to understand and navigate**
- 🎨 **Visually appealing and modern**
- ⚡ **Efficient** in completing tasks like booking and payment
- 🔄 **Consistent** across all screens

## 🖥️ UI Screens Overview

The design consists of **9 interconnected screens** forming a complete application flow:

1. **Home Page** — Featured hotels, categories, and navigation
2. **Hotel Details Page** — Images, description, and pricing
3. **Login Page** — Secure user authentication
4. **Register Page** — New user account creation
5. **Booking Page** — Selected hotel + booking information
6. **Payment Page** — Payment details and transaction completion
7. **Confirmation Screen** — Booking/account creation success
8. **Dashboard** — Overview of user activities and bookings
9. **Bookings Page** — All past and upcoming bookings

Each screen uses **proper alignment, spacing, and visual hierarchy** to ensure readability and usability, with consistent color schemes and typography enhancing the overall visual appeal.
